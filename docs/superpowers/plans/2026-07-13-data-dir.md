# EV::WebKit data_dir Option Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `data_dir` (and optional `cache_dir`) constructor option so an `EV::WebKit` instance persists and restores its full session -- cookies, `localStorage`, `IndexedDB`, cache -- to an isolated directory.

**Architecture:** One wiring change in `new()`: when `data_dir` is given, construct the `NetworkSession` as `WebKit::NetworkSession->new($data_dir, $cache_dir)` (real paths) instead of `->new(undef, undef)` (WebKit's shared default). Everything else -- forcing non-ephemeral, composing with `cookie_jar`, the two croaks -- is guard logic around that one line. No new files, no new dependencies.

**Tech Stack:** Perl 5.10+, WebKitGTK 6.0 via Glib::Object::Introspection, `File::Spec::Functions::rel2abs` (already imported).

- [ ] **Step 3: Wire `data_dir` into the session construction**

In `lib/EV/WebKit.pm`, change the `$ephemeral` decision and the `NetworkSession` construction. Replace:

```perl
    my $ephemeral = $o{cookie_jar} ? 0 : (defined $o{ephemeral} ? $o{ephemeral} : 1);
    my $session = $self->{session} = $ephemeral
        ? WebKit::NetworkSession->new_ephemeral
        : WebKit::NetworkSession->new(undef, undef);
```

with:

```perl
    # cookie_jar OR data_dir forces a non-ephemeral session: WebKit's
    # set_persistent_storage (cookie_jar) and its on-disk storage (data_dir)
    # both bail out for an ephemeral session.
    my $ephemeral = ($o{cookie_jar} || defined $o{data_dir})
        ? 0 : (defined $o{ephemeral} ? $o{ephemeral} : 1);
    # data_dir points the whole session -- cookies, localStorage, IndexedDB,
    # cache -- at $data_dir, isolated from every other instance, and restored on
    # the next construction with the same path. cache (regenerable) defaults to a
    # subdirectory; cache_dir overrides it. rel2abs because a path relative to
    # WebKit's cwd is a footgun (cwd drifts) -- same reason pdf() uses it. WebKit
    # creates the leaf dirs; the parent must exist and be writable.
    my $session = $self->{session} =
        $ephemeral            ? WebKit::NetworkSession->new_ephemeral
      : defined $o{data_dir}  ? WebKit::NetworkSession->new(
                                    rel2abs($o{data_dir}),
                                    rel2abs($o{cache_dir} // "$o{data_dir}/cache"))
      :                         WebKit::NetworkSession->new(undef, undef);
```

- [ ] **Step 4: Add the POD**

In `lib/EV/WebKit.pm`, immediately after the `=item C<< jar_format => 'sqlite' | 'text' >>` block (the paragraph ending `Ignored unless C<cookie_jar> is given.`), insert:

```pod
=item C<< data_dir => $path >>

Points this instance's entire session -- cookies, C<localStorage>,
C<IndexedDB>, the HTTP cache, service-worker state -- at C<$path>, and restores
it whenever an instance is later built with the same C<$path>. Two instances
with different C<data_dir>s share nothing. Forces a non-ephemeral session (see
C<ephemeral> above); C<< data_dir => ..., ephemeral => 1 >> croaks.

Unlike C<cookie_jar>, which persists only cookies with an expiry, C<data_dir>
persists everything the browser stores, including session cookies. The one
exception is C<sessionStorage>, which WebKit treats as inherently per-session
and never writes to disk.

C<data_dir> and C<cookie_jar> compose: C<data_dir> persists everything to its
own locations, and if C<cookie_jar> is also given, cookies additionally go to
that specific queryable C<sqlite> file. A relative C<$path> is resolved against
the current directory at construction time. The parent directory must exist and
be writable; WebKit creates C<$path> itself.

B<One live instance per C<data_dir> at a time.> WebKit's C<localStorage> and
C<IndexedDB> databases are not built for concurrent writers, so two live
instances pointed at the same C<data_dir> in one process can corrupt them --
the same caution as not sharing a file between C<cookie_jar> and
C<save_cookies>.

=item C<< cache_dir => $path >>

Overrides where C<data_dir>'s disposable cache is written (default:
C<< $data_dir/cache >>). Useful for putting the regenerable cache on C<tmpfs>,
or keeping a backed-up C<data_dir> free of it. Ignored -- and a croak -- unless
C<data_dir> is given, since a cache dir with no data dir would leak cache to
WebKit's shared location and defeat the isolation.
```

## Persisting a full session

`cookie_jar` persists cookies. To persist a whole session -- `localStorage`,
`IndexedDB`, the cache, a login that lives in any of them -- pass
`data_dir => $path`:

    my $b = EV::WebKit->new(data_dir => "$ENV{HOME}/.myapp/session");
    # ...log in...
